# MarkOS-AgentFlow Enterprise v2.0

> Next-generation multi-agent software delivery platform with a high-end Frosted Glass UI.

## 🌟 Overview
MarkOS-AgentFlow is an enterprise-grade orchestration platform that automates the software delivery lifecycle (PM -> Dev -> QA) using specialized AI agents. It features a robust state machine and a highly visual dashboard.

## 🏗 Architecture
- **Backend**: FastAPI, SQLModel, PostgreSQL, Redis.
- **Frontend**: Next.js 14 (App Router), Tailwind CSS, Zustand.
- **Orchestrator**: A specialized control loop for agent coordination and handoff.

## 🚀 Quick Start

### 1. Clone the repository
```bash
git clone https://github.com/mktt-ai-global/MarkOS-AgentFlow.git
cd MarkOS-AgentFlow
```

### 2. Configure Environment
```bash
cp .env.example .env
# Edit .env with your credentials
```

### 3. Start with Docker (Recommended)
```bash
docker-compose up --build
```
- **Dashboard**: http://localhost:3000
- **API Docs**: http://localhost:8000/docs

## 📖 API Documentation
The API documentation is automatically generated by FastAPI and is available at `/docs` when the server is running. It follows the OpenAPI specification.

## 🛠 Development Workflow

### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt -r requirements-dev.txt
uvicorn app.main:app --reload
```

### Frontend
```bash
cd frontend
npm install
npm run dev
```

## ⚠️ Troubleshooting
- **Database Connection**: Ensure the `DATABASE_URL` in `.env` matches your Docker or local PostgreSQL setup.
- **Port Conflict**: If ports 3000 or 8000 are occupied, adjust mappings in `docker-compose.yml`.
- **Next.js Build**: If fonts or images fail to load, ensure `next/font` is properly configured.
